# Contributing

欢迎提交 issue、改进建议和 pull request。

为了保持这个项目轻量、易用，提交改动时建议遵循以下原则：

- 优先保持单文件可运行的使用体验
- 尽量不引入额外依赖和构建工具
- 优先修复真实使用场景中的问题
- 文档与代码变更尽量同步更新

## 建议的贡献方向

- 会议邀请文本解析规则扩展
- 交互细节和移动端体验优化
- 浏览器兼容性修复
- 可访问性改进
- 文档补充与示例完善

## 提交流程

1. Fork 本仓库并创建分支
2. 完成修改并自测
3. 提交清晰的 commit message
4. 发起 pull request，说明变更目的和影响范围

## 本地验证

这是一个纯前端静态页面项目，通常只需要：

1. 用浏览器打开 `index.html`
2. 验证新增、编辑、删除、搜索、快速入会是否正常
3. 验证剪贴板导入与主题切换是否正常
4. 验证桌面端和移动端布局是否正常
